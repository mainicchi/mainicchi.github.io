My basic project workflow

Understand

Gain knowledge of users, context, technologies, and the brand. The problem of the user has to be identified to later start working towards the solutions. Understanding the mission and goals is crucial to get the right answers.

Research

Gathering data by 1:1 interviewing, focus groups, surveys and usability testing for example and to base assumptions on real life evidence/data.

Analyze

Build user personas (these act as guides) on gathered data. Make user journeys and user scenarios to see the bigger picture.

Design

Create site maps, user flows and wireframes these are low-fidelity mock-ups. From these start building a prototype that takes shape from the collected data

Launch

Evaluate design with the users to obtain feedback and develop the product in the design team. Continuous flow of feedback is needed to launch the right product.

Evaluate

Investigate where the process went right and if the solutios really solved the problem of the users. With this knowledge, future projects will run even more smoothly.
